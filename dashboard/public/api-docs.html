INVENTORY API · VERSION 1

库存明细 API 接口文档

给其他网站的服务端读取当前商品、价格和库存状态。接口只读取脱敏快照， 不会触发额外店铺请求。

GET Bearer API Key 120 次/分钟/IP

01

Endpoint

接口地址

业务请求使用 HTTPS GET；浏览器跨域预检支持 OPTIONS。

GET https://103.193.172.111.sslip.io/stock-monitor/api/v1/inventory
内容类型application/json
更新频率约 5 分钟
历史下架不返回
02

Authentication

身份验证

在请求头中携带独立的 API Key。网页登录密码和登录 Cookie 不能代替 API Key。

HTTP Header
Authorization: Bearer <API_KEY>
不要把 API Key 写进网页前端

请保存在网站后端、Vercel Function、Cloudflare Worker 或服务器环境变量中。浏览器源码和开发者工具无法保密。

03

Quick start

调用示例

cURL

Shell
curl --fail --silent \
  -H "Authorization: Bearer <API_KEY>" \
  https://103.193.172.111.sslip.io/stock-monitor/api/v1/inventory

Node.js / 网站后端

JavaScript
const response = await fetch(
  "https://103.193.172.111.sslip.io/stock-monitor/api/v1/inventory",
  {
    headers: {
      Authorization: `Bearer ${process.env.LDXP_INVENTORY_API_KEY}`,
    },
  },
);

if (!response.ok) {
  throw new Error(`Inventory API returned ${response.status}`);
}

const inventory = await response.json();
console.log(inventory.items);

05

Errors

错误码

HTTPerror处理建议
401unauthorized检查 Authorization 请求头和 API Key。
403origin_rejected浏览器来源未加入白名单；前端通常表现为 CORS 或网络错误。
405method_not_allowed业务请求方法不是 GET。
429rate_limited降低频率后重试；代理层 429 可能不是 JSON，且不一定包含 Retry-After。
503snapshot_stale监控已停止或快照过期，30 秒后重试，不要展示旧库存。
503snapshot_unavailable监控尚未完成首次轮询，或快照暂时无法读取。
06

Security

跨域、限流与数据边界

  • ✓

    推荐由调用网站的后端请求；这种方式不需要配置 CORS。

  • ✓

    浏览器直连需要配置精确 HTTPS 域名白名单，不支持携带网页登录 Cookie。

  • ✓

    每个来源 IP 每分钟最多 120 次，超限返回 429。

  • ✓

    只读取脱敏后的监控快照，不返回代理、Telegram、店铺令牌、访客 ID 或历史下架商品。
